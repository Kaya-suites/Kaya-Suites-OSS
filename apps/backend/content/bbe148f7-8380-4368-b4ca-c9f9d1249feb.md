## 2. System Loan Process

All new SE employees are eligible for a company-issued laptop/equipment loan. Follow these steps:

### How to Start a System Loan

1. **Submit a Request** – File a system loan request via [IT Portal / HR System – *update link here*].
2. **Select Equipment** – Choose from available hardware options (e.g., MacBook Pro, ThinkPad, external monitors).
3. **Approval** – Your request will be reviewed and approved by your direct manager and the IT department.
4. **Pickup / Delivery** – Once approved, coordinate with IT for equipment pickup or delivery.
5. **Sign the Loan Agreement** – Review and sign the System Loan Agreement form, which covers:
   - Equipment specifications and serial numbers
   - Duration of loan (typically for the length of employment)
   - Care and return policy
   - Liability for damage or loss
6. **Asset Tagging** – IT will register the equipment under your name in the asset management system.

### Return Policy
- Equipment must be returned in working condition upon separation or end of loan period.
- Contact IT at least **5 business days** before your last day to schedule the return.

## 3. Development Environment Setup

### 3.1 Go (Golang) Development Setup

Kaya Suites backend services primarily use **Go (Golang)**. Follow these steps to set up your local development environment:

#### Install Go
1. Download the latest stable version from [https://go.dev/dl/](https://go.dev/dl/).
2. Follow the installation instructions for your OS (macOS, Linux, or Windows).
3. Verify the installation:
   ```bash
   go version
   ```

#### Configure Your Workspace
1. Set your `GOPATH` (if not using Go modules default):
   ```bash
   export GOPATH=$HOME/go
   export PATH=$PATH:$GOPATH/bin
   ```
2. Clone the main repository:
   ```bash
   git clone [repo URL – update here]
   cd [project-directory]
   ```
3. Install dependencies:
   ```bash
   go mod tidy
   ```

#### Recommended IDE / Editor
- **GoLand** (JetBrains) – Full-featured Go IDE
- **VS Code** with the **Go extension** (`golang.go`)

#### Running the Application Locally
```bash
go run main.go
```
Or using the project Makefile (if available):
```bash
make run
```

#### Running Tests
```bash
go test ./...
```
